# KeyJam 产品需求文档(PRD) — 打字即作曲

- 版本:v0.2(黑客松 demo 阶段)
- 更新:2026-06-14
- 状态:开发中 · 单人 · 2 天 demo

---

## 1. 一句话定位

**随便敲键盘就是在作曲——你不可能弹错。** 字母键映射到五声音阶旋律、每次按键自动吸附到节拍上、背景有和弦和鼓点兜底。把电脑递给任何人,乱敲都自成音乐。

## 2. 背景与问题

普通人想"创作音乐 / 感受演奏"的门槛极高:要学乐器、懂乐理、练技术。现有工具要么太专业(DAW)、要么太被动(播放器)、要么太随机(纯随机音符没有音乐性)。

**机会**:把"作曲"降维成"打字",让系统保证正确性(音阶锁定 + 节拍吸附 + 和弦伴奏),用户只需要有节奏感——甚至不需要节奏感(16 分音符吸附兜底)。

## 3. 目标用户与场景

- **主要用户**:没学过乐器、不懂乐理、但喜欢音乐、想要"创造感/解压"的普通人。
- **典型场景**:
  - 一个人戴着耳机,随便敲键盘,听着自己"即兴"的音乐放松。
  - 把笔记本递给朋友:"你乱敲试试",对方敲出来的东西竟然真好听——这就是社交货币。

## 4. 北极星

把"创作音乐"的门槛降到零——**敲键盘就是作曲,乱敲也好听**。

## 5. 核心体验(当前 MVP)

1. 用户**点击开始**(或按任意键),系统启动音频引擎。
2. 用户**随便敲字母键**:
   - 键盘三排映射到三个八度的**五声音阶**——怎么敲都不会跑调。
   - 每下按键**自动吸附到最近的 16 分音符**——乱敲也踩拍。
3. **背景伴奏**自动循环:和弦进行 + 鼓点,提供和谐的音乐上下文。
4. **视觉反馈**:背景极光 + 按键亮起 + 扩散光环 + 飘升光球,按音高着色。
5. **多种音乐风格**可选:lo-fi / EDM / 爵士 / 氛围等,一键切换整个伴奏和音色。

> 设计哲学:**防错 Jam(Error-proof Jam)**——系统把"可能犯错"的维度全部锁死(音阶、节奏、和声),用户只负责"什么时候敲、敲多快"这种不可能错的决定。

## 6. 功能需求

| 编号 | 优先级 | 功能 | 说明 |
|---|---|---|---|
| FR-1 | P0 | 键盘-音阶映射 | 字母键三排映射到 C 大调五声音阶(C D E G A)三个八度;下排低音、中排中音、上排高音 |
| FR-2 | P0 | 节奏吸附 | 每次按键落到下一个 16 分音符,确保旋律踩在拍上 |
| FR-3 | P0 | 和弦伴奏 | 四小节循环(Cmaj7 → Am7 → Fmaj7 → G7),铺底 pad 音色 |
| FR-4 | P0 | 鼓点伴奏 | 底鼓 + 军鼓 + 踩镲的 lo-fi 鼓 pattern |
| FR-5 | P0 | 旋律音色 | PolySynth 三角波 + 混响 + 延迟,柔和钟琴/电钢感 |
| FR-6 | P0 | 视觉特效 | 极光背景 + 按键辉光 + 扩散光环 + 飘升光球,按音高着色 |
| FR-7 | P1 | 多风格切换 | lo-fi / EDM / 爵士 / 氛围四种风格,一键切换音阶/和弦/鼓/BPM/音色/视觉 |

## 7. 非功能需求

- **延迟**:按键到发声的延迟应不可感知(<20ms + 最多一个 16 分音符的吸附延迟)。
- **正确性**:随机连续快速敲击任意字母键 30s,不应出现明显跑调或不和谐感。
- **运行**:纯前端,依赖 Tone.js(CDN);无需服务器(可由任意静态托管打开,包括 `file://` 经简单调整)。
- **可达性**:仅需键盘;支持触屏(屏幕键盘可点击)。

## 8. 技术方案概要

- **音频引擎**:Tone.js 14.8.49(CDN),PolySynth → 混响 + 延迟效果链,MembraneSynth/NoiseSynth 鼓。
- **音阶逻辑**:C 大调五声音阶 `["C","D","E","G","A"]`,三排键盘映射三个八度。
- **节奏吸附**:`Tone.Transport.nextSubdivision("16n")` 确保每个音符落到最近的 16 分音符上。
- **视觉**:Canvas 2D + `globalCompositeOperation: "lighter"` 叠加模式,极光漂移 + 粒子系统。

## 9. 四种音乐风格(路线图)

| # | 风格 | BPM | 音阶 | 和弦进行 | 鼓点特征 | 音色 |
|---|------|-----|------|---------|---------|------|
| 1 | lo-fi | 78 | C 大调五声 | Cmaj7 Am7 Fmaj7 G7 | boom-bap,鬆散 | 三角波+混响 |
| 2 | EDM | 128 | A 小调五声 | Am F C G | 四拍底鼓,clap | 锯齿波+滤波 |
| 3 | 爵士 | 100 | C 布鲁斯 | Dm7 G7 Cmaj7 A7 | 刷子军鼓,ride | FM 电钢 |
| 4 | 氛围 | 60 | C 大调五声 | Cmaj7 Em7 Am7 Fmaj7 | 极简,间隔远 | 正弦 pad+长混响 |

## 10. 范围边界(明确不做)

- 不做录音/导出(MVP 不涉及)。
- 不做 MIDI 设备支持。
- 不做自定义音阶/和弦编辑(MVP 用预设)。
- 不做多人在线。

## 11. 路线图

- **现在(MVP)**:键盘五声音阶 + lo-fi 伴奏 + 极光视觉。
- **P1**:四种风格切换;触屏键盘优化;背景动画随风格变化。
- **P2**:自定义音阶/调式;录音/回放;更多视觉主题。

## 12. 风险与未决问题

- **风格切换复杂度**:每种风格需要独立的音阶、和弦、鼓、音色配置,开发和维护成本随风格数量线性增长。
- **Tone.js CDN 依赖**:海外 CDN 可能间歇性不可用,需考虑国内镜像或本地打包。
- **移动端兼容**:Tone.js 在移动 Safari 上 AudioContext 启动需要用户手势,当前已处理但需持续验证。

## 13. 成功指标

- **Demo 现场**:递给没学过音乐的人,对方敲 30s 键盘后露出"卧槽这竟然是我弹的"的表情。
- **社交传播**:有人主动掏出手机拍屏幕发朋友圈/微信群:"你们试试这个"。
