# KeyJam 产品需求文档(PRD)

- 版本:v0.1(黑客松 demo 阶段)
- 更新:2026-06-14
- 状态:开发中 · 单人 · 2 天 demo

---

## 1. 一句话定位

**把笔记本变成一把"自动挡吉他"——键盘就是扫弦。** 用户载入一首自己拥有的歌,系统把它按小节切段,用户每按一下键就"弹"出一段,用敲击的节奏去演奏这首歌的律动。

## 2. 背景与问题

普通人想"演奏 / 跟着玩"一首喜欢的歌,门槛很高:要会乐器、要识谱、要练。市面上的方案要么太专业(DAW、谱),要么只是被动听(播放器)。

**机会**:绝大多数人没有乐器、但天天用键盘。如果把"演奏"降维成"踩着节奏按键",让正确性由系统保证,就能让没有任何乐理基础的人也获得"我在演奏这首歌"的参与感。

## 3. 目标用户与场景

- **主要用户**:没有乐器基础、但喜欢音乐、想要"参与感/解压"的普通人。
- **典型场景**:
  - 一个人对着自己喜欢的歌,跟着节奏敲键盘"弹"一遍,边弹边哼。
  - 把笔记本递给朋友,对方不用学就能立刻上手玩(现场社交)。

## 4. 北极星

把"演奏 / 参与一首歌"的门槛降到最低——**你只要有节奏感,就能玩**;正确性(段落、和弦)由系统兜底。

## 5. 核心体验(当前 MVP)

1. 用户**载入一首自己拥有的音频**(上传 / 拖拽 / 内置示例曲)。
2. 可选**「去人声」**,得到伴奏轨(便于跟唱)。
3. 系统**自动估计 BPM、按小节切段**(可手动微调)。
4. 用户**每按一下空格/键 = 播放下一段**;快慢由用户掌控。
5. 视觉反馈:**6 根吉他弦被拨动**、波形条上当前段高亮 + 播放头、实体**键帽下沉**、当前段号显示。

> 设计隐喻:**笔记本=吉他,键盘=扫弦**。UI 走"乐器/硬件"质感(暖琥珀+espresso 配色,Fraunces/思源宋/Space Mono,吉他弦背景,实体键帽)。

## 6. 功能需求

| 编号 | 优先级 | 功能 | 说明 |
|---|---|---|---|
| FR-1 | P0 | 音频载入 | 上传 / 拖拽 / 内置原创示例曲;校验格式,拒绝 `.ncm` 等加密/非标准格式并给友好提示 |
| FR-2 | P0 | 自动切段 | 估计 BPM(`web-audio-beat-detector`,失败回退默认值)→ 按小节切段;BPM ± 手动微调实时重切 |
| FR-3 | P0 | 按键触发 | 空格 / 任意字母键 / 点击键帽 → 播放下一段并前进,循环;立即发声、跟手 |
| FR-4 | P0 | 去人声 | 立体声做中置消除(L−R)得到伴奏轨;单声道给出提示;可开关切回原曲 |
| FR-5 | P0 | 可视化 | 吉他弦拨动、波形条切段/高亮/播放头、键帽下沉、当前段号 |
| FR-6 | P1 | 备选·和弦弹唱 | 后端在**公共版权曲库**搜索出和弦谱 + 支持粘贴 ChordPro 谱;Web Audio 合成扫弦(已实现,当前非主线) |
| FR-7 | P2 | 保存 / 分享 | 保存一次"演奏"或配置,可分享 |

## 7. 非功能需求

- **隐私**:用户音频**只在本地浏览器内处理**,不上传服务器。
- **运行**:纯前端 + 零运行依赖的 Node 后端(仅静态托管 + 可选接口);Node ≥ 18;现代浏览器 Web Audio。
- **性能**:按键到发声延迟可忽略;切段/去人声为载入时一次性计算。
- **可达性**:键盘 + 鼠标/触摸均可触发;响应式布局。

## 8. 关键约束:版权与合规(本产品的核心约束)

- **只用用户合法拥有的内容**:音频文件由用户自带;不抓取、不下载、不解密(如网易云 `.ncm`)受版权保护的音乐。
- **不爬谱站、不自动复现版权歌词/吉他谱**:和弦谱只来自"用户粘贴"或"自维护的公共版权曲库"。
- **去人声**仅对用户自有文件做本地处理,且去除人声反而降低歌词版权暴露。
- **结论**:对主流流行歌,"只输个名字就自动出准确谱/原轨"在合法前提下不可得——**"准确"与"零摩擦自动"二选一**;正式商用需走授权(分离 API / 和弦数据 API)。

## 9. 技术方案概要

- **前端** `index.html`:单文件,Web Audio(解码 / 中置消除 / 按段播放)+ Canvas(吉他弦 + 波形)+ `web-audio-beat-detector`(CDN)估 BPM。
- **后端** `server.js`:零依赖 Node,静态托管 +(备选)`/api/song` 公共版权曲库搜索、`/api/chords` LLM 和弦(闲置)。
- **资源**:`assets/demo-loop.wav`(脚本 `scripts/make-demo.js` 合成的原创免版税示例曲)、`assets/songs.json`(公共版权曲库)。
- 工程规范见 `CLAUDE.md`、`README.md`。

## 10. 范围边界(明确不做)

- 不做受版权音乐的下载 / 解密 / 再发布。
- 不做谱站爬取、不自动复现版权歌词。
- MVP 不做高质量人声分离(用浏览器中置消除占位)。
- 不引入前端框架 / 打包器(保持单文件简单)。

## 11. 路线图

- **现在(MVP / demo)**:音频段触发 + 中置消除去人声 + 自动切段 + 乐器风 UI。
- **P1**:高质量人声分离(后端 Demucs/Spleeter 或付费 API);手动打点分段;保存/分享;扩充公共版权曲库。
- **P2**:悬浮插件形态——边在 Word / 写代码时打字边演奏,角落一个"打字就工作、停下就休息"的辉光小宠物。

## 12. 风险与未决问题

- **方向收敛**:产品在"和弦弹唱"与"音频段触发"之间反复切换,需尽快锁定主线。
- **分段准确度**:自动 BPM 在复杂混音上不稳,依赖手动 ± 兜底。
- **去人声质量**:中置消除是卡拉OK级,会削掉中置贝斯/鼓并变单声道;要更好需上 Demucs/API(成本)。
- **版权**:正式产品的音频/谱/歌词都需授权,demo 阶段靠"用户自带 + 公共版权 + 原创示例"。

## 13. 成功指标

- **Demo 现场**:能用"自己的一首歌"完整跑通;递给陌生人能立刻上手玩;评委记得住"键盘=吉他"这个点。
- **后续(若继续)**:留存(用户是否反复来玩)、单次时长、分享率。
