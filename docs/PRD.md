# KeyJam 产品需求文档(PRD) — 方案二:打字即作曲

- 版本:v0.2(黑客松 demo 阶段)
- 更新:2026-06-14
- 状态:开发中

---

## 1. 一句话定位

**随便敲键盘就是在作曲——你不可能弹错。** 字母键映射到五声音阶旋律、每次按键自动吸附到节拍上、背景有和弦和鼓点兜底。把电脑递给任何人，乱敲都自成音乐。

## 2. 背景与问题

普通人想"创作音乐 / 感受演奏"的门槛极高:要学乐器、懂乐理、练技术。现有工具要么太专业(DAW)、要么太被动(播放器)、要么太随机(纯随机音符没有音乐性)。

**机会**:把"作曲"降维成"打字"，让系统保证正确性(音阶锁定 + 节拍吸附 + 和弦伴奏)，用户只需要有节奏感——甚至不需要节奏感(16 分音符吸附兜底)。

## 3. 目标用户与场景

- **主要用户**:没学过乐器、不懂乐理、但喜欢音乐、想要"创造感/解压"的普通人。
- **典型场景**:
  - 一个人戴着耳机，随便敲键盘，听着自己"即兴"的音乐放松。
  - 把笔记本递给朋友:"你乱敲试试"，对方敲出来的东西竟然真好听——社交货币。

## 4. 北极星

把"创作音乐"的门槛降到零——**敲键盘就是作曲，乱敲也好听**。

## 5. 核心体验

1. 用户**点击开始**(或按任意键)，系统启动音频引擎。
2. 用户**随便敲字母键**:键盘三排映射到三个八度的**五声音阶**——怎么敲都不会跑调。每下按键**自动吸附到最近的 16 分音符**——乱敲也踩拍。
3. **背景伴奏**自动循环:和弦进行 + 鼓点，提供和谐的音乐上下文。
4. **视觉反馈**:背景极光 + 按键亮起 + 扩散光环 + 飘升光球，按音高着色。
5. **四种音乐风格**:lo-fi / EDM / 爵士 / 氛围，一键切换。

> 设计哲学:**防错 Jam(Error-proof Jam)**——系统把"可能犯错"的维度全部锁死(音阶、节奏、和声)，用户只负责"什么时候敲、敲多快"。

## 6. 功能需求

| 编号 | 优先级 | 功能 | 说明 |
|---|---|---|---|
| FR-1 | P0 | 键盘-音阶映射 | 字母键三排映射到 C 大调五声音阶(C D E G A)三个八度 |
| FR-2 | P0 | 节奏吸附 | 每次按键落到下一个 16 分音符 |
| FR-3 | P0 | 和弦伴奏 | 四小节循环，铺底 pad 音色 |
| FR-4 | P0 | 鼓点伴奏 | 底鼓 + 军鼓 + 踩镲 pattern |
| FR-5 | P0 | 旋律音色 | PolySynth + 混响 + 延迟 |
| FR-6 | P0 | 视觉特效 | 极光 + 按键辉光 + 扩散光环 + 飘升光球 |
| FR-7 | P1 | 多风格切换 | lo-fi / EDM / 爵士 / 氛围，一键切换 |

## 7. 四种风格

| # | 风格 | BPM | 和弦进行 | 鼓点 | 音色 | 氛围 |
|---|------|-----|---------|------|------|------|
| 1 | Lo-fi | 78 | Cmaj7 Am7 Fmaj7 G7 | boom-bap | 三角波+重混响 | 松弛慵懒 |
| 2 | EDM | 128 | Am F C G | 四拍底鼓 | 锯齿波+轻混响 | 能量亢奋 |
| 3 | Jazz | 100 | Dm7 G7 Cmaj7 A7 | 切分鼓 | 三角波+中混响 | 优雅 sophisticated |
| 4 | Ambient | 60 | Cmaj7 Em7 Am7 Fmaj7 | 极简 | 正弦波+长混响 | 空灵梦幻 |

## 8. 非功能需求

- **延迟**:按键到发声不可感知(<20ms + 最多一个 16 分音符吸附延迟)。
- **正确性**:随机快速敲击任意字母键 30s，不应出现明显跑调或不和谐。
- **运行**:纯前端，依赖 Tone.js(CDN)；可由任意静态托管打开。

## 9. 技术方案

- **音频引擎**:Tone.js 14.8.49(CDN)，PolySynth → 混响 + 延迟效果链，MembraneSynth/NoiseSynth 鼓。
- **音阶逻辑**:C 大调五声音阶 `["C","D","E","G","A"]`，三排键盘映射三个八度，所有风格共用。
- **节奏吸附**:`Tone.Transport.nextSubdivision("16n")`。
- **风格切换**:`dispose()` 旧引擎 → 按新参数重建 → 重启 Transport。
- **视觉**:Canvas 2D 极光 + 粒子系统。

## 10. 范围边界(明确不做)

- 不做录音/导出、MIDI 设备、自定义音阶/和弦编辑(MVP)。
- 不做多人在线。

## 11. 路线图

- **现在(MVP)**:键盘五声音阶 + lo-fi 伴奏 + 极光视觉 + 四种风格切换。
- **P1**:Maestro Stickman（火柴人音乐家）v1
- **P2**:音乐同步系统 + 乐队系统
- **P3**:成长系统 + 自定义音阶/录音/回放

## 12. Maestro Stickman（火柴人音乐家）

### 12.1 一句话定位

**当用户创作音乐时，一个火柴人音乐家陪伴用户一起演奏。** 它不是宠物，是音乐搭子。

### 12.2 核心设计原则

- ❌ 不要让用户觉得"我在养一只电子宠物"
- ✅ 让用户觉得"这个火柴人是我的音乐搭子"
- 它陪伴的是创作，不是生活——与普通桌宠形成差异化，与 KeyJam 音乐键盘天然品牌关联

### 12.3 角色设计

```
大圆头 60% + 细四肢身体 40%
极简 Stickman，保证小尺寸下依然可识别
```

### 12.4 MVP 功能

| 功能 | 触发条件 | 行为 |
|------|---------|------|
| **Typing Animation** | 用户打字 | 慢速→轻轻弹琴🙂 / 中速→正常演奏😄 / 快速→激情演奏🤩 |
| **Idle System** | 停止输入 0-5s | 发呆😐 |
| | 停止输入 5-20s | 伸懒腰🥱 |
| | 停止输入 20s+ | 睡觉😴 |
| | 随机 | 看四周 / 打拍子 / 玩乐器（避免完全静止） |
| **点击互动** | 点击角色 | 随机：鼓掌👏 / 鞠躬🙇 / 炫技🎹 / 跳跃😆 / 惊讶😲 |
| **乐器切换** | 用户设置 | Piano（默认）/ Guitar / Violin / Drum，火柴人演奏姿势随之变化 |

### 12.5 表情系统（纯头部表达）

```
^ ^ 开心    ★ ★ 兴奋    － － 睡觉
> < 生气    O O 惊讶    • • 专注
```

### 12.6 产品架构

```
Character (Stickman)
    +
Instrument (Piano / Guitar / Violin / Drum)
    +
Emotion (Happy / Focused / Sleepy / Excited / Surprised)
    +
State (Typing / Idle / Sleeping / Celebrating / Interacting)
```

### 12.7 V2: 音乐同步系统

不只看打字速度，而是分析实际生成的音乐：

| 音乐特征 | 火柴人反应 |
|---------|-----------|
| 节拍 | 身体跟着节奏摆动 |
| 和弦变化 | 切换演奏动作 |
| 音乐风格切换 | 自动匹配乐器和情绪 |
| Piano 生成 | 弹钢琴 |
| Guitar 生成 | 弹吉他 |
| Drum 生成 | 打鼓 |

### 12.8 V3: 乐队系统

从一个角色升级为乐队：

```
🥁 鼓手 + 🎹 钢琴手 + 🎸 吉他手 + 🎻 贝斯手
```

用户创作时整个乐队一起演奏。

### 12.9 V4: 成长系统

创作时间越长，等级越高：

```
Lv1 街头艺人 → Lv2 音乐学生 → Lv3 乐队成员 → Lv4 音乐大师 → Lv5 Maestro
```

解锁：新乐器、新动作、新表情、新服装。

### 12.10 与 KeyJam 的关联

- KeyJam 是"用户创作音乐的乐器"
- Maestro 是"陪伴用户创作的伙伴"
- 两者形成品牌矩阵：一个让你能创作，一个陪你创作
